# Safe Actions Security Review

Package 4B.5 reviews and hardens the Package 4B safe action executor before any higher-risk actions are considered.

## Threat Model

Safe actions still accept user-influenced targets from the UI, so the main risks are:

- opening an external URL instead of a local UI/path;
- command injection through a shell-like target;
- treating a network share as a local path;
- accidentally launching a target executable;
- allowing UI code to bypass policy;
- losing audit visibility for denied or failed attempts.

## Policy Boundary

The UI never executes safe actions directly. It creates an `ActionRequest`, asks `PolicyEngine` for an `ActionPlan`, shows confirmation, then calls the backend execution command.

`jsentinel_execute_safe_action` uses `SafeActionExecutor`, which plans the request again before execution. This second backend-side plan is the enforcement point. Dangerous or unsupported actions return denied/unsupported results and are written to local audit history.

## `reveal_path` Validation

Accepted targets:

- existing local file paths;
- existing local directory paths.

Rejected targets:

- empty targets;
- embedded null characters;
- leading/trailing whitespace;
- `http://`;
- `https://`;
- `file://`;
- `ms-settings:`;
- `shell:`;
- `cmd:`;
- `powershell:`;
- `javascript:`;
- raw command names such as `cmd.exe`, `powershell.exe`, and `pwsh.exe`;
- UNC/network paths such as `\\server\share`;
- paths that do not exist;
- paths that cannot be canonicalized.

Execution uses a hardcoded `explorer.exe` invocation with arguments generated by code. JSentinel does not call `cmd /C`, PowerShell, or a shell-constructed command string. It does not create, delete, move, rename, execute, or change permissions on the target.

## `open_windows_settings` Allowlist

Only exact allowlisted Windows Settings URIs are accepted:

- `ms-settings:privacy`
- `ms-settings:privacy-microphone`
- `ms-settings:privacy-webcam`
- `ms-settings:appsfeatures`
- `ms-settings:startupapps`
- `ms-settings:network-status`
- `ms-settings:windowsdefender`

Rejected examples include external URLs, `file://` URLs, command names, malformed settings URIs, shell URIs, JavaScript URIs, whitespace-padded values, and embedded null characters.

## Audit Behavior

Every backend execution attempt produces a local `ActionResult` and attempts to write it to SQLite `action_history`.

Statuses used by the safe executor:

- `succeeded`: a validated safe action was launched;
- `failed`: validation or OS launch failed;
- `denied`: policy or allowlist denied the action;
- `unsupported`: the action or platform is unsupported.

There is no upload, telemetry, analytics, account, ad SDK, or external network path for action history.

## Still Disabled

This review does not enable:

- process kill;
- firewall block/unblock;
- startup disable/restore;
- quarantine move/restore;
- delete-on-reboot;
- force unlock;
- registry writes;
- file watcher;
- mic/camera monitor;
- automatic mitigation.

Before any dangerous action is enabled, it needs a separate design, policy review, confirmation copy, rollback/audit model, and CI coverage.
